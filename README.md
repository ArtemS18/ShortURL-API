# ShortURL API — Высоконагруженный сервис сокращения ссылок

Проект представляет собой отказоустойчивый, масштабируемый API-сервис для создания уникальных коротких ссылок. В основе генерации идентификаторов лежит кастомный алгоритм **Snowflake ID (Base63)**, обеспечивающий генерацию уникальных строк фиксированной длины без обращения к централизованным хранилищам или базам данных.

---

##  Быстрый запуск

### Локальная разработка (Development)

Запуск одной ноды приложения и базы данных PostgreSQL в docker-контейнерах с пробросом портов напрямую на хост-машину ($8000$).

```bash
make dev

```

* **API эндпоинты:** `http://localhost:8000`
* **Swagger:** `http://localhost:8000/docs/index.html`
* **PostgreSQL:** `localhost:5434`

### Продакшен окружение (Production)

Развертывание отказоустойчивого кластера из **3 независимых нод бэкенда**, базы данных PostgreSQL и балансировщика нагрузки **Nginx** в качестве Reverse Proxy.

```bash
make prod

```

* **Точка входа (Nginx балансировщик):** `http://localhost:8001`
* **Swagger:** `http://localhost:8001/docs/index.html`
* **PostgreSQL:** `localhost:5435`

### Тестирование и покрытие кодом

```bash
# Запуск всех юнит и интеграционных тестов
make test

# Анализ покрытия кода (тестами покрыты слои delivery, repository, usecase)
make cover

```

## Архитектура и дизайн-решения

Сервис спроектирован с учетом требований к высокой пропускной способности и консистентности данных.
Для удовлетворения ТЗ (**строго 10 символов**, алфавит `A-Za-z0-9_`)  используется **кастомная битовая маска распределенного ID (59 бит)**, транслируемая в **Base63**:

* **Битовая схема (59 бит):**
* `TimestampBits (41 бит)`: гарантирует **69 лет** работы сервиса без переполнения от кастомной эпохи (`2026-01-01`).
* `NodeBits (8 бит)`: позволяет горизонтально масштабировать бэкенд до **256 независимых инстансов (нод)**.
* `SequenceBits (10 бит)`: защищает от коллизий при высокой нагрузке, позволяя генерировать до **1024 уникальных ссылок в миллисекунду**.


## Справочник команд (Makefile)

| Команда | Описание |
| --- | --- |
| `make run` | Локальный запуск Go-приложения напрямую на хосте (требует внешнюю БД) |
| `make dev` | Поднятие окружения разработки (`shorturl-dev`: 1 бэкенд + PG) |
| `make prod` | Поднятие продакшен кластера (`shorturl-prod`: Балансировщик Nginx + 3 Бэкенда с уникальными `SRV_NODE_ID` + PG) |
| `make swag` | Автогенерация Swagger-документации на основе аннотаций в коде |
| `make test` | Очистка кэша тестов и запуск всего тестового пакета |
| `make cover` | Запуск тестов для слоев delivery/repository/usecase с выводом процента покрытия |

## Переменные окружения

Каждая нода бэкенда параметризуется следующими переменными (передаются через docker-compose):

* `SRV_NODE_ID`: Уникальный номер инстанса бэкенда (0-255). Используется алгоритмом Snowflake для обеспечения глобальной уникальности ID в кластере без межсерверной синхронизации.
* `DATABASE_TYPE`: Тип используемой бд (`postgres` или `in-memory`).
* `PG_HOST` / `PG_PORT`: Параметры подключения к пулу PostgreSQL.
* `SRV_BASE_URL`: Базовый префикс для формирования возвращаемой клиенту короткой ссылки.
